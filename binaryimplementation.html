Binary Search

Implementation


Binary search is implemented using following steps...

Consider the following list of elements and the element to be searched...

                             

Step 1 : Read the search element from the user.

                            Search element = 31.


Step 2 : Find the middle element in the sorted list.
  mid = low + (high - low) / 2

    Here it is, 0 + (9 - 0 ) / 2 = 4 (integer value of 4.5). So, 4 is the mid of the array.

                             


Step 3 : Compare the search element with the middle element in the sorted list.

    Now we compare the value stored at location 4, with the value being searched, i.e. 31


Step 4 : If both are matched, then display "Given element is found!!!" and terminate the function.


Step 5 : If both are not matched, then check whether the search element is smaller or larger than the middle element.

     We find that the value at location 4 is 27, which is not a match. As the value is greater than 27 and we have a sorted array, so we also know that the target value must be in the upper portion of the array

                             


Step 6 : If the search element is larger than middle element, repeat steps 2, 3, 4 and 5 for the right sublist of the middle element.

     We change our low to mid + 1 and find the new mid value again.
     low = mid + 1
     mid = low + (high - low) / 2

     Our new mid is 7 now. We compare the value stored at location 7 with our target value 31

                             
     The value stored at location 7 is not a match, rather it is more than what we are looking for.


Step 7 : If the search element is smaller than middle element, repeat steps 2, 3, 4 and 5 for the right sublist of the middle element.

     The value 7 is more than what we are looking for.So,the value must be in the lower part from this location

                             

    Hence, we calculate the mid again. This time it is 5

                             

    We compare the value stored at location 5 with our target value. We find that it is a match.

                             

    We conclude that the target value 31 is stored at location 5.

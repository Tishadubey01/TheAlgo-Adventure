Selection Sort

Algorithm - Analysis


 Algorithm:

procedure selectionSort
 list: array of items
 n: size of list
 for i=1 to n-1
// Set current element as minimum
  min=i

// Check the element to be minimum
  for j=i+1 to n
   if list[j] < list[min] then
    min=j
   end if
  end for
//Swap the minimum element with current element
  if indexMin != i the
   swap list[min] and list[i]
  end if
 end for
end procedure

 Pseudo Code

void selectionSort(int *Arr, size_t length)
{
 for (size_t i=0;i<length;i++)
 {
  size_t smallestindex=i;
// run through the unsorted elements in the input, finding the smallest one.
  for (size_t j=i+1;j<length;j++)
   {
    if (Arr[j]<Arr[smallestindex])
    {
     smallestindex=j;
    }
   }
// move the smallest element to the front of the unsorted portion of the input (swapping with whatever's already there)..
  int temp= Arr[smallestindex];
  Arr[smallestindex]=Arr[i];
  Arr[i]=temp;
 }
}

 Time Complexity Analysis:

Selection Sort requires two nested for loops to complete itself, one for loop is in the function selectionSort, and inside the first loop we are making a call to another function smallestindex, which has the second(inner) for loop.

Hence for a given input size of n, following will be the time and space complexity for selection sort algorithm:

  1. Best Case Time Complexity: O(n²)
  2. Worst Case Time Complexity: O(n²)
  3. Average Time Complexity: O(n²)


The time complexity of selectionSort is O(n²) in all the 3 cases (worst, average and best) as this algorithm sorts an array or list by repeatedly finding the minimum value.

Space Time Complexity:

  1. Selection sort is an in-place algorithm.
  2. It performs all computation in the original array and no other array is used .
  3. Hence, the space complexity works out to be O(1).

Linear Search

Algorithm - Analysis


 Algorithm:

procedure linear_search (list, value)
 for each item in the list
   if match item == value
    return the item's location
   end if
 end for
end procedure

 Pseudo Code

int linear_search(int values[], int target, int n)
{
  for(int i = 0; i < n; i++)
  {
   if (values[i] == target)
   {
    return i;
   }
  }
 return -1;
}

 Time Complexity Analysis:

Linear Search time complexity analysis is done in the following ways:-

Best Case [ O(1) ]
 In the best possible case,

  1. The element being searched may be found at the first position.
  2. In this case, the search terminates in success with just one comparison.
  3. Thus in best case, linear search algorithm takes O(1) operations.


Worst Case [ O(n) ]
 In the worst possible case,

  1. The element being searched may be present at the last position or not present in the array at all.
  2. In the former case, the search terminates in success with N comparisons.
  3. In the later case, the search terminates in failure with N comparisons.
  4. Thus in worst case, linear search algorithm takes O(N) operations.


Average Case [ O(N) ]
 Let there be N distinct numbers: a1, a2, ..., a(N-1), aN. We need to find element P.
There are two cases:

  1. Case 1: The element P can be in N distinct indexes from 0 to N-1.
  2. Case 2: There will be a case when the element P is not present in the list.

There are N case 1 and 1 case 2. So, there are N+1 distinct cases to consider in total.If element P is in index K, then Linear Search will do K+1 comparisons.
Number of comparisons for all cases in case 1 = Comparisons if element is in index 0 + Comparisons if element is in index 1 + ... + Comparisons if element is in index N-1
=1 + 2 + ... + N
= N * (N+1) / 2 comparisons
If element P is not in the list, then Linear Search will do N comparisons.
Number of comparisons for all cases in case 2 = N. Therefore,

  1. Total number of comparisons for all N+1 cases = N * (N+1) / 2 + N
    = N * ((N+1)/2 + 1)
  2. Average number of comparisons = ( N * ((N+1)/2 + 1) ) / (N+1)
    = N/2 + N/(N+1)

The dominant term in "Average number of comparisons" is N/2.
Thus in the Average Case, linear search algorithm takes O(N) operations.

Space Time Complexity:

The linear search only need one element (that is the variable you used to compare with all the elements in the array), so no matter how big or small your array is, the algorithm only consumes the same amount of memory for all input.
Thus is space complexity is O(1)
